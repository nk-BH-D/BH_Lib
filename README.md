# Инструкция по пользованию Telegram-ботом

Бот предназначен для хранения и получения решений задач, разделённых по курсам.

Система ролей: USER / ADMIN / ROOT.

🔑 Общие положения

• Регистрация происходит автоматически при нажатии /start

• Данные хранятся в PostgreSQL

• Все действия логируются

• Для администраторов и root используется пароль

• Пароль хранится в виде SHA-256 хеша

• Пароль может содержать только:

    - латинские буквы (A–Z, a–z)
    - цифры (0–9)
    - символы '_' '#' '@' '-'
    
• Сессия ввода пароля — 1 минута

## 👤 USER (обычный пользователь)

### Регистрация

    1. Нажать /start
    2. Пользователь автоматически регистрируется как user
    3. Пароль не требуетс

### Возможности USER

    • Получение решений задач
    • Просмотр истории своих запросов

#### Получение решения

Формат сообщения:

    <курс>
    <название_задачи> 

Если решение существует:

    • Бот вернёт текст решения
    • Запрос будет записан в историю пользователя

Если решения нет:

• Сообщение: Решения такой задачи нету в базе

#### Просмотр истории запросов

• Через соответствующую кнопку меню

• Отображается:

    - список запросов 
    - общее количество 

### Ограничения

❌ нельзя добавлять решения

❌ нельзя удалять решения

❌ нельзя обновлять решения

❌ нет доступа к администрированию

## 🛠 ADMIN (администратор)

### Назначени

    • Назначается только ROOT 
    • После назначения должен нажать /start и установить пароль 

### Возможности ADMIN

• Все возможности USER

• Добавление решений

• Обновление своих решений

• Удаление своих решений

• Просмотр статистики

#### ➕ Добавление решения

Формат сообщения:

    <курс>;;<название_задачи_1>;;<условие_1>;;<решение_1>;;<название_задачи_2>;;<уловие_2>;;<решение_2>...

Важно:

    • ;; — строгий разделитель
    • Можно добавлять несколько задач за раз

Если решение уже существует:

    • Добавление будет отклонено

#### ✏ Обновление решения

Формат сообщения:

    <курс>;;<название_задачи>;;<условие>;;<новое_решение>

⚠ Обновлять можно только решения, добавленные этим администратором

#### ❌ Удаление решения

Формат сообщения:

    <курс>;;<название_задачи>;;<условие>;;<точный_текст_решения> 

⚠ Удалить можно только своё решение. Если решение добавлено другим администратором — будет отказ.

#### Примечание

Eсли условие не трубеться оставьте это поле пустым

Примеры:

Для добавления:

    <курс>;;<название_задачи_1>;;;;<решение_1>;;<название_задачи_2>;;;;<решение_2>...
    
Для обновления:

    <курс>;;<название_задачи>;;;;<новое_решение>
    
Для удаления:

    <курс>;;<название_задачи>;;;;<точный_текст_решения>
    
## 👑 ROOT (владелец системы)

### Инициализация

• ROOT определяется по ROOT_ID из переменных окружения

• При первом /start:

    - задаёт пароль 
    - регистрируется как root 

### Возможности ROOT

• Все возможности ADMIN

• Назначение администраторов

• Удаление администраторов

#### ➕ Назначение администратора

Формат команды:

    <user_id> <chat_id>

Результат:

    • Пользователь получает статус admin
    • Ему приходит уведомление
    • Он должен нажать /start и задать пароль

#### ❌ Удаление администратора

Формат команды:

    <user_id> <chat_id> 

Результат:

    • Статус меняется на user
    • Пароль администратора сбрасывается
    • Пользователь получает уведомление

## Команды в Telegram

### /docx

Позволяет получить ссылку на GitHub репозиторий проекта

### /start

• Запускает или перезапускает чат с ботом

• Позваляет изменить пароль ROOTам и ADMINам
